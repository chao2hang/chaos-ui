# Internationalization (i18n)

## Overview

Chaos UI uses `react-i18next` + `i18next` for internationalization. Translation resources are organized by namespace.

## Supported Locales

- `zh-CN` (Simplified Chinese) — Default
- `en-US` (English) — Full
- `ja-JP` (Japanese) — Planned
- `ko-KR` (Korean) — Planned

## Namespace Structure

```
lib/i18n/resources/
  zh/
    common.json
    ui.json
    error.json
    data.json
    chart.json
    cookie.json
    language.json
    marketing.json
    mobile.json
    navigation.json
    notification.json
    tour.json
    transfer.json
    upload.json
  en/
    (same structure)
```

## Usage

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation("common");
  return <p>{t("save")}</p>;
}
```

## Linting

The `@chaos/no-hardcoded-chinese` ESLint rule detects hardcoded Chinese strings. Currently set to `warn` for business components and `error` for UI components.

## Migration Guide

1. Replace all hardcoded Chinese strings with `t('key')` calls
2. Add translation keys to the appropriate namespace JSON files
3. Run `npm run lint` to verify no hardcoded Chinese remains
4. Once all strings are extracted, upgrade the lint rule from `warn` to `error`

## Number/Date/Currency Localization

Use `Intl.NumberFormat`, `Intl.DateTimeFormat`, and `@/lib/format` utilities:

```tsx
import { formatDate, formatCurrency } from "@/lib/format";

formatDate(new Date(), "zh-CN"); // 2026年1月15日
formatDate(new Date(), "en-US"); // 1/15/2026
formatCurrency(1234.56, "CNY"); // ¥1,234.56
formatCurrency(1234.56, "USD"); // $1,234.56
```
