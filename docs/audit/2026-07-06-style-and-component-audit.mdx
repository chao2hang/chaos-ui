### 0.1 技术栈与分层

| 维度 | 现状 |

## A 类：严重运行时 Bug

### A1 — `--font-sans` 自引用死循环

- **现象**：`--font-sans: var(--font-sans)` 自引用，CSS 变量解析为无效值（fallback 链断裂），依赖 `font-sans` utility 的元素回退到浏览器默认字体（衬线），与 Geist Sans 设计意图不符。
- **根因**：`@theme inline` 中把 `--font-sans` 映射到 `var(--font-sans)` 自身，而非 Next.js 注入的 `--font-geist-sans`。`inline` 关键字会在 `:root` 展开引用，形成无限自引用。
- **文件:行**：
  - `styles.css:758` — `--font-sans: var(--font-sans);`
  - `apps/docs/app/globals.css:16` — 同（第 18 行 `--font-heading` 连带失效）
  - `packages/chaos-design-ui/styles.css:10` — 同（第 12 行 `--font-heading` 连带失效）
  - ⚠️ **核验修正**：计划描述「影响 4 个 globals」，但 `app/globals.css` 仅 3 行 `@import`（`tailwindcss` / `../styles.css` / `../styles/scrollbar.css`），本身不重复声明 `--font-sans`，其字体继承自被 import 的根 `styles.css`。**实际真源为 3 处**。
- **修复建议**：三处 `--font-sans: var(--font-sans)` 改为 `--font-sans: var(--font-geist-sans);`（与 `--font-mono: var(--font-geist-mono)` 对齐）。
- **难度**：🟢 低。**优先级**：**P0**。

### A2 — `--radix-popover-trigger-width` 在 base-ui 项目中不被注入

- **现象**：`Autocomplete` / `Mentions` 的下拉面板宽度与触发器不一致（面板回退到内容宽度或默认宽度），破坏「面板宽度 = 触发器宽度」的视觉契约。
- **根因**：项目 popover 已从 Radix 迁移到 base-ui，base-ui 不注入 `--radix-popover-trigger-width` 变量，该 `var()` 解析为空。
- **文件:行**：
  - `components/ui/autocomplete.tsx:135` — `className="w-[var(--radix-popover-trigger-width)] p-1"`
  - `components/ui/mentions.tsx:124` — 同
  - ⚠️ **核验修正**：计划提到「子包镜像」，核验 `packages/chaos-design-ui` 未见独立的 `autocomplete.tsx` / `mentions.tsx` 含该变量（子包 re-export 根实现）。**修复点集中在根 `components/ui/` 2 处**。
- **修复建议**：改用 base-ui 提供的 `var(--anchor-width)`，并提供 fallback：`w-[var(--anchor-width,100%)] p-1`。修改前需运行时确认当前 base-ui 版本注入的精确变量名。
- **难度**：🟡 中。**优先级**：**P0**。

### A3 — docs 站缺动画 token / keyframes

- **现象**：docs 站中依赖 `data-[state=open]:animate-in` / `data-[state=closed]:animate-out` 的组件（Dialog / Popover / Tooltip 等）无过渡动画。
- **根因**：`apps/docs/app/globals.css:3` 把 `@import "tw-animate-css"` 注释掉；且该 globals 用 `@import "tailwindcss" source(none)` 独立入口，不 import 根 `styles.css`，未继承 keyframes / `--animate-*` 段。
- **文件:行**：`apps/docs/app/globals.css:3`（`/* @import "tw-animate-css"; */`）。
- **修复建议**：二选一——(a) 取消第 3 行注释；(b) 让 docs globals `@import "../../../styles.css"` 复用根源；(c) 在 docs `@theme` 补齐 keyframes。推荐 (a)。
- **难度**：🟢 低。**优先级**：**P0**。

### A4 — docs 站缺 `styles/scrollbar.css`

- **现象**：docs 站滚动条为浏览器默认样式，与根 Storybook / 主应用的自定义滚动条不一致。
- **根因**：根 `app/globals.css:3` 通过 `@import "../styles/scrollbar.css"` 引入；`apps/docs/app/globals.css` 无此 import，且 `apps/docs/styles/` 目录不存在。
- **文件:行**：`apps/docs/app/globals.css`（缺 import）；根对照 `app/globals.css:3`。
- **修复建议**：docs globals 增加 `@import "../../../styles/scrollbar.css";`（推荐 re-export 根侧真源，避免漂移）。
- **难度**：🟢 低。**优先级**：**P0**。

### A5 — 根 `app/layout.tsx` 未接入 `next-themes` ThemeProvider

- **现象**：根 Next.js app（`app/`）无法运行时切换暗色模式；`.dark` class 仅在 docs 站可切换。
- **根因**：`apps/docs/app/layout.tsx` 已包裹 `ThemeProvider`，根 `app/layout.tsx` 未引入。
- **文件:行**：`app/layout.tsx`（grep `ThemeProvider` / `next-themes` 均无命中）。
- **修复建议**：在根 `app/layout.tsx` 包裹 `<ThemeProvider attribute="class" defaultTheme="system" enableSystem>`；或明确「根 app 仅作 landing 无需主题切换」并在 README 备注。
- **难度**：🟢 低。**优先级**：P0（若根 app 演示暗色）/ P2（若仅 landing）。

## D 类：设计系统层缺失 token（应有但全无）

> 以下 token **在 `@theme` 命名空间中完全缺失**（既未在 `:root` 定义，也无 Tailwind utility 映射），导致组件层只能硬编码 Tailwind 默认值，设计系统层无法统一调优。

| 类目 | 缺失内容 | 影响 |
| --- | --- | --- |
| **Spacing scale** | `--spacing-*`（0.5 / 1 / 2 / 4 / 8 …）在 `@theme` 未注册 | 依赖 Tailwind 默认 `spacing`（0.25rem 步进），无法整套换算（如切换 8px baseline） |
| **字号 utility 映射** | `--text-xs / sm / base / lg / xl / 2xl …` 在 `@theme` 未注册 | 组件层硬编码 `text-sm` / `text-lg` 且无法通过 token 调整 |
| **line-height / letter-spacing** | 无 token；`styles.css` base 层 h1–h6 硬编码 | 排版一致性依赖各组件自觉，回退风险高 |
| **z-index 在 `@theme` 注册** | `:root` 有 `--z-index-*`（8 层）但**未映射到 `@theme --z-*`**，Tailwind utility `z-*` 不识别 | 组件多用 `z-50` 硬编码而非 `z-modal` / `z-toast` |
| **Border width** | 无 `--border-width-*` token；组件硬编码 `border` / `border-2` | 无法整体切换粗细基准 |
| **Focus ring 指标** | 无 `--focus-ring-width` / `--focus-ring-offset` / `--focus-ring-color` 统一 token | 各组件 `focus-visible:ring-*` 各自为战，a11y 一致性差 |
| **数据可视化语义 token** | 无 `--chart-positive` / `--chart-negative` / `--chart-grid` / `--chart-axis` / `--chart-tooltip-bg`（仅 `--chart-1..5`） | 财务 / 监控图表无法通过 token 表达语义 |
| **Density token** | 无 `--density-compact-*` / `--density-comfortable-*` | 无法整体切换紧凑 / 舒适模式 |
| **Motion 命名空间** | `--duration-*` / `--ease-*` 在 `:root` 定义但**未接入 `@theme --animate-*`**（见 E）；无 `--motion-collapse-in` / `--motion-fade-in` 高阶命名 | 组件多用 `transition-all duration-200` 硬编码 |
| **`--radius-full` 派生** | 无独立 `--radius-full`（当前依赖 Tailwind `rounded-full = 9999px`） | 圆角 token 谱系不完整 |
| **`.dark` 显式覆盖缺失** | `.dark` 中未显式覆盖 `--disabled-*` / `--duration-*` / `--ease-*`（这些 token 亮暗色应可差异化） | 暗色模式下禁用态 / 动效感知一致，缺少差异化空间 |

**建议批次**：**P1** — 补 `--spacing-*` / `--text-*` / focus ring / chart 语义色（4 大类）；**P2** — density / motion 高阶命名 / border-width / radius-full；**P3** — `.dark` 差异化覆盖。

## E 类：未消费的 Dead Token

> 以下 token **已在 `:root` 定义**，但**未在 `@theme` 命名空间映射**为 Tailwind utility，组件代码全部改用硬编码 Tailwind utility（如 `text-sm` / `z-50` / `duration-200`），导致 token 闲置。

| Token 前缀 | 定义位置 | 现状 |
| --- | --- | --- |
| `--font-size-*` | `styles.css` `:root` | 已定义但无 `--text-*` @theme 映射 → 组件用 `text-sm` / `text-base` 硬编码 |
| `--ease-*` | `styles.css` `:root` | 已定义但无 `@theme --ease-*` 映射 → 组件用 `ease-in-out` 硬编码 |
| `--duration-*` | `styles.css` `:root` | 已定义但无 `@theme --duration-*` 映射 → 组件用 `duration-200` / `duration-300` 硬编码 |
| `--z-index-*` | `styles.css` `:root` | 已定义但无 `@theme --z-*` 映射 → 组件用 `z-50` / `z-[9999]` 硬编码 |

- **后果**：token 层已有变量但**不可用**，形成"面子工程"；未来任何设计系统调优（如整体降低动效时长）无法通过修改 token 实现。
- **修复建议**：在 `styles.css` `@theme inline` 段增加对应映射：
  ```css
  @theme inline {
    --text-xs: var(--font-size-xs);
    --text-sm: var(--font-size-sm);
    /* ... */
    --duration-fast: var(--duration-fast);
    --ease-in-out: var(--ease-in-out);
    --z-modal: var(--z-index-modal);
    --z-toast: var(--z-index-toast);
    /* ... */
  }
  ```
- **难度**：🟢 低（一次性 mapping）。**优先级**：**P1**。
