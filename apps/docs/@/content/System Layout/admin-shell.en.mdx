# AdminShell Admin Shell

> System-level layout shells for admin pages.

AdminShell — a system-level layout shell for admin/dashboard/auth pages.

## When to Use

Use AdminShell when you need: AdminShell — a system-level layout shell for admin/dashboard/auth pages. Use for admin shells (sider, tabs, header) to keep multi-page navigation consistent.

## Notes

- **Client Component**: uses `"use client"` and React hooks; render on the client.

### Height chain (CUI-LAYOUT-04)

- Root classes are **`h-full min-h-0`**: the shell **fills its parent**, not a default `min-h-svh`. On short pages the sider/shell shrink to content height unless an ancestor sets a definite height.
- Unlike **AuthLayout**, which defaults to `min-h-svh` (`fill="viewport"`), AdminShell still expects the host to cooperate.
- Recommended Next.js App Router root layout:

<CodeBlock
  code={`// app/layout.tsx — Next.js App Router
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html className="h-full">
      <body className="min-h-svh h-full">
        {/* Theme / auth Providers must also pass height (h-full / min-h-svh / flex) */}
        {children}
      </body>
    </html>
  )
}`}
  lang="tsx"
/>

- Escape hatches: mount under a full-height ancestor, or pass `className="min-h-svh"` for a local viewport fill. A future `fill="viewport" | "parent"` API (aligned with AuthLayout) is an optional enhancement, not part of this contract.
