# AdminShell Admin Shell

> 系统布局 / System-level layout shells for admin pages.

Admin Shell — 系统级布局外壳,用于后台/仪表盘/认证等整页布局。
AdminShell — a system-level layout shell for admin/dashboard/auth pages.

## 何时使用 / When to Use

当你需要「Admin Shell」相关能力时使用：Admin Shell — 系统级布局外壳,用于后台/仪表盘/认证等整页布局。用于后台壳层与系统级布局（侧栏、页签、顶栏）；保证多页应用导航一致。

Use AdminShell when you need: AdminShell — a system-level layout shell for admin/dashboard/auth pages. Use for admin shells (sider, tabs, header) to keep multi-page navigation consistent.

## 注意事项 / Notes

- **Client Component**: 使用 `"use client"`,依赖 React hooks/状态,需在客户端渲染。

### 高度链 / Height chain（CUI-LAYOUT-04）

- 根节点为 **`h-full min-h-0`**：**填满父级**，不是默认 `min-h-svh`。短内容页上侧栏/壳高度 ≈ 内容高，除非祖先给出确定高度。
- 与 **AuthLayout** 不同：认证页默认 `min-h-svh`（`fill="viewport"`）更自洽；Admin 壳仍要求宿主配合。
- Next.js App Router 根布局推荐：

<CodeBlock
  code={`// app/layout.tsx — Next.js App Router
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html className="h-full">
      <body className="min-h-svh h-full">
        {/* Theme / auth Providers must also pass height (h-full / min-h-svh / flex) */}
        {children}
      </body>
    </html>
  )
}`}
  lang="tsx"
/>

- 逃生舱：在全高祖先内挂载，或给壳加 `className="min-h-svh"` 做本地视口填充。未来若提供 `fill="viewport" | "parent"` 将另开 enhancement，与 AuthLayout 对齐。
