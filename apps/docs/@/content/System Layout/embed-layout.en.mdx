# EmbedLayout Embed Layout

> System layout

Full-height frame for third-party hosts (iframe / card / drawer): fills the **parent height** (`h-full`) by default, optional header + scrollable body.

## When to Use

Use when embedding a business view into a fixed-height host and you need a sticky top bar with an independently scrolling body.

## Examples

### Basic Usage

<CodeBlock code={`import { EmbedLayout } from "@chaos_team/chaos-ui/layout"

export function Basic() {
  return (
    <div className="h-[480px] rounded-lg border">
      <EmbedLayout header={<div className="px-4 py-2 text-sm font-semibold">Order detail</div>}>
        <div className="p-4 text-sm">Body content…</div>
      </EmbedLayout>
    </div>
  )
}`} lang="tsx" />

## API / Props

### EmbedLayout

| Prop        | Type              | Default | Description                                                  |
| ----------- | ----------------- | ------- | ------------------------------------------------------------ |
| `className` | `string`          | —       | Root class; override height (e.g. `h-svh` for viewport-full) |
| `header`    | `React.ReactNode` | —       | Optional top bar (inside `<header>`)                         |
| `children`  | `React.ReactNode` | —       | Body (`flex-1` + `overflow-auto`)                            |

## Notes

- **Height comes from the host**: root defaults to `h-full`, not `h-screen`. The host (or `className`) must set a definite height or the layout collapses.
- For browser viewport height: `className="h-svh"` or `h-screen`.
- **Client Component**: uses `"use client"`.
