# ConfirmDialog 确认对话框

> 业务组件

确认对话框 — 面向 ERP/企业场景的业务组件。

## 何时使用

需要确认对话框功能时使用本组件。


## 代码示例

### 基础用法

<CodeBlock code={`import { ConfirmDialog, ConfirmDialogContainer } from "@/components/business/confirm-dialog"

export function Basic() {
  return (
    <ConfirmDialog>
      <ConfirmDialogContainer>Content</ConfirmDialogContainer>
    </ConfirmDialog>
  )
}`} lang="tsx" />


## API / Props

### ConfirmDialog

| Prop | Type | Default | 说明 |
|---|---|---|---|
| `className` | `string` | — | 自定义样式类名 |
| `open` | `boolean` | — | 是否打开(受控) |
| `onOpenChange` | `() => void` | — | 打开状态变化回调 |
| `title` | `string` | `确认操作` | 标题 |
| `description` | `React.ReactNode` | `此操作不可撤销，是否继续？` | 描述文本 |
| `confirmText` | `string` | `确认` |  |
| `cancelText` | `string` | `取消` |  |
| `variant` | `"default" \| "destructive"` | `default` | 样式变体 |
| `onConfirm` | `() => void` | — |  |
| `loading` | `boolean` | — | 是否加载中 |
| `icon` | `React.ReactNode` | — | 图标元素 |
| ...rest | `React.HTMLAttributes` | — | 透传原生 DOM 属性 |

## 注意事项

- **Client Component**: 使用 `"use client"`,依赖 React hooks/状态,需在客户端渲染。
- **复合组件**: 包含 1 个子组件(ConfirmDialogContainer),使用组合模式按需导入。
