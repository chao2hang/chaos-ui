# Drawer 抽屉

Chaos UI 抽屉组件，基于 vaul，从屏幕边缘滑出的模态面板。
A modal panel that slides in from the screen edge, built on vaul.

## 何时使用 / When to use

用于侧边模态面板：设置面板、详情面板、多步骤表单、筛选面板等。与 Dialog 的区别：Drawer 侧边滑出，适合长内容或多步骤操作。

Use for side modal panels: settings, details, multi-step forms, filter panels. Unlike Dialog (centered modal), Drawer slides from the edge — best for longer content.

## API

### Drawer

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| defaultOpen | `boolean` | `false` | 默认是否打开 | Default open state |
| open | `boolean` | — | 受控：是否打开 | Controlled open state |
| onOpenChange | `(open: boolean) => void` | — | 打开状态变化回调 | Callback on open state change |
| dismissible | `boolean` | `true` | 点击遮罩是否关闭 | Allow closing on overlay click |
| ...rest | `React.ComponentProps<typeof DrawerPrimitive.Root>` | — | 透传 vaul 属性 | Passthrough vaul props |

### DrawerContent

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| className | `string` | — | 自定义样式类名 | Custom CSS class |

### 其他子组件

| 组件 | Props | 描述 |
|---|---|---|
| DrawerTrigger | `React.ComponentProps<typeof DrawerPrimitive.Trigger>` | 打开抽屉的触发器 |
| DrawerClose | `React.ComponentProps<typeof DrawerPrimitive.Close>` | 关闭抽屉的触发器 |
| DrawerOverlay | `React.ComponentProps<typeof DrawerPrimitive.Overlay>` | 遮罩层 |
| DrawerHeader | `React.ComponentProps<'div'>` | 标题和描述的容器 |
| DrawerTitle | `React.ComponentProps<typeof DrawerPrimitive.Title>` | 抽屉标题 |
| DrawerDescription | `React.ComponentProps<typeof DrawerPrimitive.Description>` | 抽屉描述 |
| DrawerFooter | `React.ComponentProps<'div'>` | 底部操作区容器 |

## 注意事项

- **vs Dialog**：Drawer 从屏幕边缘滑出（默认底部），适合长内容和多步骤操作。Dialog 居中弹出，适合短内容确认。移动端优先使用 Drawer。
- **dismissible**：设为 `false` 时点击遮罩不会关闭，适用于必须通过按钮确认才能关闭的场景（如未保存提示）。
- **无障碍**：基于 vaul（Radix Vaul），遵循 WAI-ARIA Dialog Pattern。Escape 关闭，焦点陷阱。DrawerTitle 和 DrawerDescription 分别关联 `aria-labelledby` 和 `aria-describedby`。
- **Portal**：Drawer 通过 Portal 渲染，不受父容器样式限制。
