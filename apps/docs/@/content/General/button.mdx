# Button 按钮

Chaos UI 按钮组件，基于 Base UI 构建，支持多种 variant、size 和语义化状态。
A Base UI-powered button component with multiple variants, sizes, and semantic states.

## 何时使用 / When to use

需要触发操作时使用按钮。支持普通操作、提交表单、链接跳转等场景。选择合适的 variant (default / destructive / outline / secondary / ghost / link) 来表达操作权重。

Use a button to trigger an action. Choose the right variant (default / destructive / outline / secondary / ghost / link) to convey action weight.

## API

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| variant | `'default' \| 'destructive' \| 'outline' \| 'secondary' \| 'ghost' \| 'link'` | `'default'` | 按钮变体风格 | Button style variant |
| size | `'xs' \| 'sm' \| 'default' \| 'lg' \| 'xl' \| 'icon-xs' \| 'icon-sm' \| 'icon'` | `'default'` | 按钮尺寸 | Button size preset |
| disabled | `boolean` | `false` | 禁用状态 | Disabled state |
| asChild | `boolean` | `false` | 作为子元素渲染（用于 Link 场景） | Render as child (e.g., for Next.js Link) |
| className | `string` | — | 自定义样式类名 | Custom CSS class |
| ...rest | `ButtonPrimitive.Props` | — | 透传 Base UI Button 原生属性 | Passthrough Base UI Button props |

## 注意事项

- **asChild 与路由**：配合 `asChild` 可将按钮渲染为 Next.js `<Link>` 或其他自定义元素，避免 `<a>` 嵌套 `<button>` 的无障碍问题。
- **无障碍**：Base UI Button 内置键盘支持 (Enter/Space 触发)。disabled 状态下自动移除 tabIndex，但为确保 screen reader 可读，建议保留可见文本。
- **Variant 语义**：`destructive` 用于删除/不可逆操作；`outline` / `ghost` 用于低权重操作；`link` 用于文本级跳转。
- **icon 尺寸**：`icon-xs` / `icon-sm` / `icon` 自动设为正方形，适合纯图标按钮场景。
