# Collapsible 折叠容器

Chaos UI 折叠容器组件，基于 Base UI Collapsible，用于按需显示/隐藏内容区域。
A Base UI-powered component for showing/hiding content regions on demand.

## 何时使用 / When to use

用于需要按需展开/折叠的单块内容区域。与 Accordion 的区别：Collapsible 是单个独立容器，无 item 概念，更轻量。

Use for a single collapsible content region. Unlike Accordion, Collapsible is a single standalone container — lighter and simpler.

## 代码示例 / Examples

<Collapsible>

    This is the collapsible content area. Put any React components here.

## API

### Collapsible

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| defaultOpen | `boolean` | `false` | 默认是否展开 | Default expanded state |
| open | `boolean` | — | 受控：是否展开 | Controlled expanded state |
| onOpenChange | `(open: boolean) => void` | — | 展开状态变化回调 | Callback on open state change |
| disabled | `boolean` | `false` | 禁用折叠功能 | Disable the collapsible |
| className | `string` | — | 自定义样式类名 | Custom CSS class |

### CollapsibleTrigger

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| className | `string` | — | 自定义样式类名 | Custom CSS class |

### CollapsibleContent

| Prop | Type | Default | Description (zh) | Description (en) |
|---|---|---|---|---|
| className | `string` | — | 自定义样式类名 | Custom CSS class |

## 注意事项

- **受控模式**：传入 `open` + `onOpenChange` 进入受控模式，适用于需要外部控制展开/折叠的场景。
- **vs Accordion**：Collapsible 是单区域折叠，Accordion 是多区域互斥/多选折叠。简单场景优先使用 Collapsible。
- **无障碍**：Base UI Collapsible 内置 WAI-ARIA 支持——Trigger 使用 `button` 角色，Content 关联 `aria-controls`。展开/折叠状态通过 `data-state` 属性反映。
- **动画**：Base UI 默认提供展开/折叠的 CSS 过渡动画，可通过覆盖 `data-state` 相关样式自定义。
