# Getting Started

Quick guide to using Chaos UI components in your project.

## Installation

```bash
pnpm install
```

## Development

```bash
# Start Storybook documentation
pnpm storybook

# Start Next.js dev server
pnpm dev

# Build for production
pnpm build

# Build Storybook for deployment
pnpm build-storybook

# Run linter
pnpm lint
```

## Basic Usage

Import a component and use it in your code:

```tsx
import { Button } from '@chaos_team/chaos-ui/ui'

export default function Page() {
  return (
    <div>
      <Button variant="default">Click me</Button>
    </div>
  )
}
```

## Component Anatomy

Every Chaos UI component follows a consistent pattern:

```tsx
import { cn } from '@chaos_team/chaos-ui/lib'

// 1. Import the component
import { Button } from '@chaos_team/chaos-ui/ui'

// 2. Use it with TypeScript props
function MyComponent() {
  return (
    <Button
      variant="default"      // Variant prop
      size="md"              // Size prop
      disabled={false}       // State prop
      onClick={() => {}}     // Event handler
    >
      Button Text
    </Button>
  )
}
```

## Styling

Chaos UI uses Tailwind CSS 4 with CSS custom properties. All components accept a `className` prop for custom styling:

```tsx
<Button className="mt-4 w-full">
  Full Width Button
</Button>
```

Use the `cn()` utility to merge class names:

```tsx
import { cn } from '@chaos_team/chaos-ui/lib'

<Button className={cn(
  "base-classes",
  isActive && "active-classes",
  className
)}>
```

## Theming

Chaos UI supports light and dark modes via CSS variables. Toggle using the theme switcher in Storybook toolbar.

### Custom Theme

Override CSS variables in your `globals.css`:

```css
:root {
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
}
```

## TypeScript

All components are fully typed. Import types directly:

```tsx
import { Button, type ButtonProps } from '@chaos_team/chaos-ui/ui'
```
