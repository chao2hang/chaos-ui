## 背景与动机

- 历史已出现「FilterBar 修了工具栏，别的工具栏仍 h-8」「SearchTable inset 修了 TreeTable 未跟」「AreaChart 首帧修了 Line 跟了但 Scatter 仍 320」类复发。
- 原则：**issue 修一点 → 模式扫一片**；审计结论进 `designs/`，状态看板可追踪。

## 审计方法

1. 取近 2 周 closed issues（#8–#48）按**缺陷模式**聚类。
2. `rg` / 脚本扫：
   - `Trigger` + `render={<div|span|a|input>` 且窗口内无 `nativeButton={false}`
   - `truncate` 且无 `title=` / `labelOverflow` 的树/列表 label
   - 业务工具栏 `h-8` vs `Button size="sm"`
   - `onExpandRow` / `hasChildren \|\| !!onExpand` 类懒加载
   - pure SVG chart `viewBox` / `320` / `ResizeObserver` / `preserveAspectRatio`
   - 选择器/行编辑器硬编码英文 UI 串
3. 人工对照「正确先例」组件（已修文件）。
