# 组件文档质量修复 设计计划

## 概述

对 Chaos UI 文档站点中所有组件的 MDX 文档进行系统性质量修复，覆盖描述文案、i18n 双语支持、代码示例质量、渲染正确性四大维度。

## 背景与动机

### 当前数据

| 指标                                             | 数值  | 占比 |
| ------------------------------------------------ | ----- | ---- |
| `components.meta.ts` 中注册的组件                | 496   | 100% |
| 含模板描述的组件（meta）                         | 22    | 4.4% |
| 含 MDX 文档的独立 slug                           | ~500+ | —    |
| i18n 完成（含 `.en.mdx` + `.zh.mdx`）            | 331   | ~26% |
| 仅含基准 bootstrap `.mdx`                        | 932   | ~74% |
| 残留 `ui-*` / `layout-*` / `business-*` 重复 MDX | 466   | —    |

### 核心问题

#### 问题 1：模板描述（22 个组件）

`components.meta.ts` 中 22 个组件使用统一模板描述，毫无信息量：

- EN: `"XXX — a general-purpose primitives used across the app. from Chaos UI."`
- ZH: `"XXX — Chaos UI 的通用基础组件。"`

影响的组件（全部在 `General` 分类）：AudioPlayer, Collapsible, Direction, Icon, InfiniteScroll, KeyboardShortcut, Masonry, Mentions, MobilePullRefresh, MobileSwipeAction, NumberTicker, Rating, SequencePreview, SignaturePad, Spin, SplitButton, Tour, Transfer, Typography, UserBrowse, VideoPlayer, WithPermission

#### 问题 2：i18n 缺失（932 个组件 = 74%）

绝大多数组件仅有单语言 bootstrap `.mdx`，缺少 `.en.mdx` 和 `.zh.mdx` 双语文件。

Bootstrap 文件内容示例（`direction.mdx`）：

```mdx
# Direction

> 通用基础组件 / General-purpose primitives used across the app.
> 适用于Direction相关场景。
> Use when you need a direction.

<CodeBlock code={`...<Direction />...`} />
```

#### 问题 3：代码示例无效

Bootstrap 模板生成的示例代码完全无意义：

- `<Direction />` — 无 children，看不到任何视觉效果
- `<Icon />` — 缺少必填 `icon` prop，TypeScript 会报错
- 不展示任何有意义的 props、variants 或真实场景

#### 问题 4：残留重复文件（466 个）

存在大量 `ui-*.mdx` / `layout-*.mdx` / `business-*.mdx` 前缀的文件，是旧架构的残留。这些文件：

- 标题格式错误（如 `# Direction Ui Direction`）
- 内容与正确的 MDX 文件重复
- 在 `mdx-loaders.ts` 中被注册，但不会被正常路由访问
- 增加维护负担和混乱

#### 问题 5：Icon 页面报错

`/components/General/icon` 页面加载时报错，可能与以下原因相关：

- `<ComponentPreview>` 尝试渲染 `<Icon>` 但没有传必填 `icon` prop
- MDX 中的 `<Icon />` 示例代码不符合 TypeScript 要求

## 设计方案

### 修复优先级

```
P0（阻塞性，先修）
├── Icon 页面报错 —— 阻止用户访问
├── Direction 页面白屏 —— 阻止用户访问
└── 所有含必填 props 但示例代码未传的组件

P1（i18n + 描述，分批修）
├── 22 个模板描述的组件 meta → 写真实描述
├── 932 个组件的 i18n 双语文件 → 运行 split-mdx-locales 自动拆分
└── 代码示例无效的组件 → 手动补写

P2（清理）
├── 466 个 ui-/layout-/business- 重复文件 → 删除
└── mdx-loaders.ts 中对应的旧条目 → 重新生成
```

### 技术方案

#### Phase 1：紧急修复（P0）

**目标**：修复 Direction 和 Icon 两个明显出问题的页面，建立基准质量。

1. **修复 `icon.mdx`**
   - 补写正确的代码示例：`<Icon icon={Search} size="lg" />` 等
   - 添加尺寸对照、spin/pulse 动画示例
   - 修复 `components.meta.ts` 中的描述

2. **修复 `direction.mdx`**
   - 补写有 children 的示例：`<Direction dir="rtl"><p>مرحبا</p></Direction>`
   - 添加 ltr/rtl 对比示例
   - 修复 `components.meta.ts` 中的描述

3. **运行 `split-mdx-locales.mjs` 对这两个文件生成 `.en.mdx` / `.zh.mdx`**

4. **验证页面可正常渲染**

#### Phase 2：批量 i18n 拆分（P1）

**目标**：利用已有工具 `split-mdx-locales.mjs` 自动拆分所有 bootstrap `.mdx` 为双语文件。

1. 运行 `node scripts/split-mdx-locales.mjs`
   - 该脚本已实现：从双语混合 MDX 自动生成 `.zh.mdx` 和 `.en.mdx`
   - 基于 CJK/ASCII 比例自动路由段落
   - 保留 imports、JSX、代码块不变

2. 重新生成 `mdx-loaders.ts`：`node scripts/generate-mdx-map.mjs`

3. 验证：随机抽查 10 个组件的页面，确认中英文显示正确

#### Phase 3：描述补写（P1）

**目标**：修复 22 个模板描述组件的 `components.meta.ts` + MDX 文件。

按组件逐个补写：

- 真实的组件功能描述（中英文）
- 何时使用场景说明
- 注意事项（来自源码分析）

**Phase 3 分批策略**（每批 5-7 个）：

| 批次 | 组件                                                                                  |
| ---- | ------------------------------------------------------------------------------------- |
| 1    | Direction, Icon, Spin, Typography, Button, Collapsible                                |
| 2    | AudioPlayer, VideoPlayer, SignaturePad, Rating, Tour                                  |
| 3    | Transfer, SplitButton, InfiniteScroll, Masonry, Mentions                              |
| 4    | NumberTicker, KeyboardShortcut, SequencePreview, MobilePullRefresh, MobileSwipeAction |
| 5    | UserBrowse, WithPermission                                                            |

#### Phase 4：代码示例补写（P1）

**目标**：确保每个组件至少有 1 个可运行的代码示例。

逐组件分析源码 props，补写：

- 基础用法示例
- 关键 variant/size 示例（如有）
- 至少覆盖 1 个真实使用场景

#### Phase 5：清理残留文件（P2）

**目标**：删除 466 个 `ui-*` / `layout-*` / `business-*` 重复 MDX。

1. 确认每个 `ui-*` 文件对应的正名文件已存在且正确
2. 批量删除重复文件
3. 重新生成 `mdx-loaders.ts`：`node scripts/generate-mdx-map.mjs`
4. 再次运行全量类型检查

## 风险与注意

1. **Business 分类组件量极大**（860 个），i18n 拆分耗时较长，建议最后处理
2. **`split-mdx-locales.mjs` 可能产生不完美的拆分** — bootstrap 模板内容少，拆分质量可接受，但描述补写后需要重新运行拆分
3. **`ComponentPreview` 容错** — 如果组件需要必填 props 但未提供，可能需要增加 fallback 渲染逻辑，而非直接崩溃
4. **不要修改 `components.meta.ts` 的结构** — 它被 `generateStaticParams` 和路由使用
5. **优先修 `General` 分类** — 这是用户最先看到的分类
