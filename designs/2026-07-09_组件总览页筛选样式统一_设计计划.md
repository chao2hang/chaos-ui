# 组件总览页样式重新设计 设计计划

## 概述

彻底重构 /components 页面的布局、样式和交互，消除三处冗余筛选入口（Hero 胶囊条、sticky 顶部胶囊条、侧边栏），统一为单一探索器组件。

## 背景与动机

- **三处筛选入口互相竞争**：Hero 区域的分类胶囊、sticky 顶部胶囊条、左侧侧边栏分类列表三者功能重叠，视觉密度不一致
- **样式 token 不统一**：激活态在各区域用了不同的 brand 透明度组合（/10 vs /15 vs /25），hover 态有的有 border 反馈有的没有
- **间距断层**：Hero 区域和下方 sticky 条之间有大段空白，sticky 条和卡片网格之间又没有清晰分隔
- **用户反馈**：标注了两处红框（左侧栏 + 顶部胶囊条），明确表示"不好看"

## 设计方案

### 核心思路

**做减法**：将三个组件（ComponentSearch + ComponentSidebar + HeroQuickStats）合并为一个统一组件 `ComponentExplorer`。

- 去掉 Hero 区域（大标题 + 描述 + 分类胶囊）
- 去掉 sticky 顶部胶囊条
- 保留左侧侧边栏，但搜索框移入侧边栏顶部
- 主内容区直接展示分类 section + 卡片网格

### 交互

- **单一搜索源**：侧边栏搜索框过滤侧边栏 + 主内容
- **GitHub 风格 / 快捷键**：按 `/` 聚焦搜索框
- **⌘K 命令面板**：保留原有全局命令面板
- **IntersectionObserver**：滚动联动高亮侧边栏当前分类
- **折叠**：分类 section header 点击折叠/展开，Business 子分组独立折叠
- **移动端**：< md 隐藏侧边栏，搜索框移至主内容顶部
