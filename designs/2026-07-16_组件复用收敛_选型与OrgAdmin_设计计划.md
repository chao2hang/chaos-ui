# 组件复用收敛：选型强化 + OrgAdmin 接 OrgTree

## 概述

审计显示：ui 原子与 `BrowseDialog` 适配器复用较好，但 **表格多栈、Picker 平行实现、OrgAdmin 自绘树** 会继续诱导 AI/人重复造轮子。本波次先做 **可执行契约 + 一个示范重构**。

## 背景与动机

- 问题：不定义清楚时 AI 乱作；定义后若存量示范仍是平行实现，规则效力有限。
- 目标：
  1. 选型表补齐 Browse/Picker/Org/表格梯子与禁止清单
  2. `OrgAdminPage` 左树复用 `OrgTree`（示范「页级组合 ui」）
  3. intro 弱化已弃用 `AdvancedDataTable`
  4. 记录 Picker/表格后续收敛 backlog（不本波次全做）

## 设计方案

### 技术方案

#### A. 文档（P0）

- 选型表增加：Browse vs Picker 定义、Org 三件套、表格梯子、full-custom picker 清单指向 backlog
- intro zh/en：数据展示去掉 AdvancedDataTable 主推，改为 SearchTable/ProTable
- designs 模板/README 已挂钩（前序完成）

#### B. OrgAdminPage → OrgTree（P1 示范）

| 项        | 做法                                                                                    |
| --------- | --------------------------------------------------------------------------------------- |
| 数据      | `OrgAdminTreeNode` → `OrgTreeNode`（count/badges/readOnly 进 `meta` + `renderNode`）    |
| 搜索      | `OrgTree searchable` + `searchPlaceholder`                                              |
| 选中      | `selectable="single"` + `selectedKeys` / `onSelect`                                     |
| 展开      | 默认展开有 children 的根；受控可后续加，本波次用 default + 搜索自动展开（OrgTree 内置） |
| 空/加载   | 外层保留 `treeLoading` / `emptyTree`；无数据时不挂 OrgTree                              |
| a11y 小补 | `OrgTree` 搜索框 `aria-label`、展开钮 Expand/Collapse（稳住测试与无障碍）               |
| API       | **不破坏** `OrgAdminPageProps` 对外形状                                                 |

## 风险与注意

- OrgTree 点击有 children 的节点会 **同时选中+切换展开**（现有 OrgTree 行为）；与旧 OrgAdmin「点标签选中、点箭头展开」略有差异，可接受为统一 ui 行为；若产品强要求分离，后续给 OrgTree 加 `expandOnSelect={false}`。
- OrgTree 空态文案固定；无数据时用页面级 `emptyTree`。
- 只读文案「只读」仍硬编码（与旧实现一致）；i18n 另波次。
