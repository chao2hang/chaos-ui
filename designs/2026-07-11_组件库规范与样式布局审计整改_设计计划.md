# 组件库规范与样式布局审计整改 设计计划

## 概述

基于 2026-07-11 深度只读审计，系统整改 Chaos UI 作为**专业组件库**时的规范落地差距与样式/布局产品化债务。

范围覆盖四层：

1. **规范诚实性** — 文档 / 导出 / ESLint / 覆盖率与代码一致
2. **S0 布局与堆叠** — Admin 壳层、z-index、GlobalLoading
3. **双实现与 API 梯子** — Mobile kit、Table、size、Button loading、deprecation warn
4. **质量门禁与主题消费** — coverage 阶梯、play 交互、axe、token 消费、peerDeps

**本计划不重复 v1.3–1.5 已完成工作**（shadow package、假 QR→真 QR、AdminShell 脱 AppShell、Tab 双实现合并、business 垫片清理）。

## 背景与动机

### 为什么需要？

项目已有完整书面规范（`CONVENTIONS.md`、`component-spec.md`、`AGENTS.md`）和大规模组件库存（UI ~183 / Business ~330 / Layout ~27 / Mobile ~27），但：

- **规范未真正绑定代码**（文档、ESLint、覆盖率、Story `play` 与现状脱节）
- **公共 API 不诚实**（主入口 = UI only，README 却示例根导入 `DataTable`）
- **样式/布局产品化债务**（Admin 壳层不一致、z-index 失控、mobile/table 双实现）
- **API 一致性不足**（size 多套词汇、DateRangePicker 双份、Loading 双概念）

### 解决什么问题？

| 严重度 | 主题              | 代表问题                                                                                |
| ------ | ----------------- | --------------------------------------------------------------------------------------- |
| HIGH   | API / 文档 / 门禁 | 主入口误导、peerDeps 过重、覆盖率虚高、0 个 play story、ESLint story 路径过时           |
| S0     | 布局              | AdminHeader `h-16` vs TopBar `h-14`；aside 小屏消失；z-index 50/100/9999 vs token 1000+ |
| S1     | 缺陷 / 重复       | GlobalLoading 双重文案；ui vs mobile pull/swipe 双栈；AdvancedDataTable 仍可发现        |
| S2     | Token             | 硬编码 zinc/hex、picker max-h、shell 宽度 magic number                                  |

### 预期效果

- 新用户按 README 安装后示例全部可用
- 无「同名不同契约」的未文档化公共导出
- AdminShell 三断点可用；遮罩堆叠可解释
- 质量门禁数字 = 文档数字；ESLint 规则指向真实路径

### 审计证据来源（只读）

- 规范：`CONVENTIONS.md`、`component-spec.md`、`AGENTS.md`、`README.md`、`ARCHITECTURE.md`
- 打包：`package.json` exports、`tsup.config.ts`
- 壳层：`components/layout/admin-shell.tsx` 等
- Loading/QR：`components/ui/global-loading.tsx`、`qrcode-display.tsx`
- Token：`styles.css`
- 历史：`docs/audit/*`、`designs/2026-07-11_deprecated与重复组件清理_设计计划.md`

## 设计方案

### Wave 1 — 规范诚实性 + 消费 API（1–2 天）

**目标**：文档与运行时一致；工具链指向真实路径；双名边界先写清。

| ID   | 任务                             | 文件 / 动作                                            | 备注                                                                       |
| ---- | -------------------------------- | ------------------------------------------------------ | -------------------------------------------------------------------------- |
| W1-1 | 修正主入口说明与示例             | `README.md`、`ARCHITECTURE.md`、`docs/architecture.md` | 根入口 = UI only；business/layout/mobile/hooks 走 subpath                  |
| W1-2 | 补全 exports 表                  | 同上 + 必要时 `package/README.md`                      | 含 `/layout`、`/mobile`、`/business/server`、`/ui-icons`                   |

| W1-4 | Token 文档对齐 `styles.css`      | `docs/design-tokens.md`、`component-spec.md` §4        | 废弃错误的 `--color-primary` / 「tokens 在 globals.css」说法               |
| W1-5 | 修复 ESLint `no-missing-story`   | `eslint-plugin-chaos/rules/no-missing-story.js`        | 扫描 `src/stories/{ui,business,layout,mobile}`                             |
| W1-6 | 评估 `require-forward-ref`       | `eslint-plugin-chaos/rules/require-forward-ref.js`     | React 19 函数组件 ref 政策：放宽 / 文档化 / 或仅对指定 form 控件强制       |
| W1-7 | 双名边界决策文档                 | 本计划「接口设计」+ 可选 `docs/` 短文                  | **DateRangePicker**、**GlobalLoading** 命名与导入路径；实现改动放 Wave 2/3 |

#### Wave 1 双名边界决策（执行前确认）

| 概念                   | 现状                                                            | 推荐决策                                                                                                                |
| ---------------------- | --------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| DateRangePicker        | ui：`[Date\|null, Date\|null]`；business：`{from,to}` + presets | **保留两套实现**，business 在文档/导出注释中标明 `value` 形状差异；2.0 考虑 rename `PresetDateRangePicker` 或统一 value |
| GlobalLoading          | ui：展示组件；business：`Provider` + `useGlobalLoading`         | **保留分层**；business 文档强调无 `GlobalLoading` 组件导出；避免跨 barrel `export *` 合并                               |
| AppShell vs AdminShell | 双壳并存                                                        | 文档「何时用谁」：AdminShell = 预接线 ERP；AppShell = 更底层拼装                                                        |

### Wave 2 — S0 布局与堆叠（2–3 天）

**目标**：壳层视觉一致、移动可用、遮罩可预测；修 GlobalLoading 明确缺陷。

| ID   | 任务                             | 文件 / 动作                                                                                                                |
| ---- | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| W2-1 | Shell 高度 / 折叠宽 token 或常量 | `styles.css` 或 layout 共享常量；`admin-header` / `top-bar` / `app-shell` / `admin-shell`                                  |
| W2-2 | 通知单一路径文档 + API 收敛      | `admin-shell` 用 NotificationCenter；`admin-header` 的 `notificationCount` 标 low-level 或 deprecate 说明                  |
| W2-3 | AdminShell 移动端 aside          | 小屏 Sheet/Drawer 回退，避免 `hidden lg:block` 直接丢内容                                                                  |
| W2-4 | z-index 政策落地                 | 映射 `--z-index-*` → utility；迁移 Dialog/Sheet/Tour/GlobalLoading/ImageViewer/business overlay；禁止新增 `z-[9999]`       |
| W2-5 | GlobalLoading 双重文案 + a11y    | `components/ui/global-loading.tsx`：Spinner 不传可见 label 或仅一处文案；补 `role="status"` / `aria-live`；z 对齐 modal 层 |
| W2-6 | 更新相关测试 / stories           | `global-loading.test.tsx`、layout stories（窄视口）                                                                        |

#### z-index 建议阶梯（与现有 token 对齐后微调）

| 层                 | 用途              | 目标                              |
| ------------------ | ----------------- | --------------------------------- |
| dropdown / popover | 菜单、选择        | token dropdown                    |
| sticky             | header / sider    | sticky                            |
| modal / sheet      | Dialog、Drawer    | modal                             |
| loading overlay    | GlobalLoading     | ≤ modal 或 = modal+1（文档写清）  |
| tour / lightbox    | Tour、ImageViewer | 最高系统层（用 token，不用 9999） |

### 接口设计

#### 公共导入约定（Wave 1 文档固化）

```typescript
// UI primitives — 根入口或 /ui
import { Button, Input, GlobalLoading, QRCode } from "@chaos_team/chaos-ui";
// 或
import { Button } from "@chaos_team/chaos-ui/ui";

// Business — 必须 subpath
import { DataTable, ProTable } from "@chaos_team/chaos-ui/business";

// Layout / Mobile / Hooks
import { AdminShell } from "@chaos_team/chaos-ui/layout";
import { MobilePullToRefresh } from "@chaos_team/chaos-ui/mobile";
import { useDebounce } from "@chaos_team/chaos-ui/hooks";

// Styles
import "@chaos_team/chaos-ui/styles.css";
```

#### GlobalLoading（Wave 2 行为契约）

```typescript
interface GlobalLoadingProps extends React.ComponentProps<"div"> {
  loading?: boolean;
  text?: string; // 仅一处可见文案；a11y 不重复播报
  fullscreen?: boolean; // default true
  spinner?: boolean; // default true
}
// 根节点：role="status" aria-live="polite"（或等价）
// fullscreen z-index：使用设计 token 层，不使用 z-[100] 任意值
```

#### Button loading（Wave 3 可选实现）

```typescript
interface ButtonProps {
  // 现有 variant / size / ...
  loading?: boolean; // true 时 disabled + 显示 Spinner，保留宽度防抖
}
```

#### size 标准草案（Wave 3）

```typescript
/** 表单与多数控件 */
type ControlSize = "sm" | "default" | "lg";
/** Spinner / 少数反馈：可保留 xs–xl，文档说明与 ControlSize 独立 */
type FeedbackSize = "xs" | "sm" | "md" | "lg" | "xl";
```

## 风险与注意

| 风险                                          | 缓解                                                                                 |
| --------------------------------------------- | ------------------------------------------------------------------------------------ |
| 文档修正暴露「根导入 business」的既有错误用法 | CHANGELOG + 迁移 snippet；不要 silent 改 tsup 把 business 塞进根（体积与 peer 爆炸） |
| z-index 全量迁移可能改变现有 app 叠层观感     | 按层 PR；Storybook 对比 Tour+Dialog+Loading                                          |
| AdminShell 移动 aside 为行为增强              | 默认开启或 opt-in prop 需在实施前定；优先默认可用                                    |
| Button loading / size 可能影响视觉 snapshot   | 有 Chromatic 时跑受影响 story                                                        |
| peerDeps optional 化可能减少强制安装          | 在 docs 写清 feature peer 表                                                         |
| business barrel 收紧为 breaking               | 独立 minor/major，勿与纯文档 Wave 混合同一 release 无说明                            |
| 工作区可能有其他未提交改动                    | 只 stage 本计划相关文件；`git add` 不用 `.`                                          |

## 优先级执行顺序（最高杠杆）
