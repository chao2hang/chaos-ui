# FE-10 桌面与移动页面范式收口 (#60 / #61) 设计计划

## 概述

一次收口桌面 FE-10 范式（#60）与移动端 ≤md 行为（#61），目标版本 **1.13.0**。两 Issue 共享 `MasterDetailLayout` / `ListPageShell` 文件，合并一个计划避免合并冲突。同时顺手修同族可一并解决的隐患，减少消费方二次提 Issue。

## 背景与动机

- 消费方 qxy-mop 将整仓迁移 ~50 个业务页（FE-10），库必须先把页面范式缺口补齐。
- 桌面：MasterDetail 高度链断 (#59 未真修) + 双竖线；ListPageShell 无 filter 时幽灵空白列；TreeCrudPage 与 Card 密度分叉；旧模板仍渲染页内 h2。
- 移动 390：OrgAdminPage 左栏挤死主区 (P0)；List 多按钮工具条裁切；MasterDetail 双汉堡；宽表无移动列策略；工作台 17/7 非响应栅格。
- 预期：库发版后消费方有唯一页面根范式答案，桌面齐高/无双线，移动 390 可作业。

## 关键决策

| 决策点                                    | 选择                                                                          | 理由                                                                                                       |
| ----------------------------------------- | ----------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| M2 `chromeBorder` 默认值                  | 默认 `true`（保留 `md:border-r`，非破坏）                                     | 避免裸 sidebar 消费方突然丢右边线；Card 子节点场景文档推荐传 `chromeBorder={false}`。additive，非 breaking |
| M1 高度链                                 | 内层 wrapper + main 加 `h-full min-h-0`                                       | 纯 bugfix，非破坏                                                                                          |
| L1 幽灵列                                 | 无 filter 时不渲染空 `flex-1`                                                 | 改进，非破坏                                                                                               |
| ListPageShell 窄屏                        | 保持默认 `end-of-filter-row`，用响应式 CSS 在 `<sm` 降级为列向 + actions 全宽 | 不改默认 prop，现有测试不变                                                                                |
| TreeCrudPage                              | 加 `sidebarWidth`（默认 320）+ 高度链 + Card 间距同源                         | API additive                                                                                               |
| OrgAdmin `<md` IA                         | 无选中→树全宽；有选中→详情全宽 + 返回钮回树                                   | 与 MasterDetail 移动一致，390 可完成「选→看」                                                              |
| MasterDetail 移动 toggle                  | 默认图标 `MenuIcon`→`FilterIcon`，aria-label 不变                             | 减少与壳层汉堡混淆；测试按 aria-label 断言不变                                                             |
| 宽表移动列                                | `ColumnDef.hideOnMobile?` additive + 文档 `fixed` 冻结策略                    | prop 可选                                                                                                  |
| `MasterListTemplate`/`MasterEditTemplate` | 仅 JSDoc `@deprecated`，不改渲染                                              | 避免破坏现存页面标题                                                                                       |

## 技术方案

### 切片 1 — MasterDetailLayout 完成态（#60 M1/M2 + #61 移动 toggle）

1. **M1 高度链**：两分支 sidebar 内层 wrapper 加 `h-full min-h-0`（`sidebarInnerStyle` 不变，在包裹 div 上加 className）；main 容器 `min-w-0 flex-1` → `min-w-0 flex-1 h-full min-h-0`。
2. **M2 双线**：新增 `chromeBorder?: boolean`（默认 `true`）。`md:border-r` 仅 `chromeBorder !== false` 时生效。
3. **移动 toggle 图标**：`MenuIcon` → `FilterIcon`，`aria-label="Toggle master"` 不变。
4. 顺手：`MasterDetailTabs` 内层补 `h-full min-h-0`（同族高度链隐患，低风险）。

### 切片 2 — ListPageShell toolbar-only + 窄屏（#60 L1 + #61）

1. **L1**：`inlineRow` 分支中，`!filterNode` 时不渲染空 `flex-1` div，actions 行直接 `justify-end`。
2. **窄屏**：filter-row 在 `<sm` 改 `flex-col`，actions `w-full justify-end`；`sm:` 恢复行内。filter 容器 `basis` 仅 `sm:` 生效。
3. 默认 `toolbarPlacement` 保持 `end-of-filter-row`（测试不变）。

### 切片 3 — TreeCrudPage FE-10 化（#60 §2）

1. 新增 `sidebarWidth?: number`（默认 `320`）。aside `w-64` → `style={{width: sidebarWidth}}` + `shrink-0`。
2. 左右 chrome 与 Card 标题行 `pb` 同源 `--card-spacing`：aside 改 `rounded-xl border bg-card py-(--card-spacing) gap-(--card-spacing)` 对齐 Card；右栏 summary 条同源。
3. `h-full min-h-0` 高度链对照 OrgAdminPage。
4. API 保留 tree/selected/onSelect/onRefresh/onCreate/query/children。

### 切片 4 — OrgAdminPage 响应式（#61 P0）

1. 内层 `<div className="flex min-h-0 flex-1 gap-3">`：`<md` 改单栏。
2. `<md` 行为：无 `selectedId` → 树全宽；有 `selectedId` → 详情全宽 + 顶部「← 返回组织树」按钮（清空选中）。
3. aside `style={{width}}` 仅 `md:` 生效；`<md` aside `w-full`。
4. 用内部 `mobileView` state（'tree' | 'detail'），选中切 detail，返回切 tree。桌面不受影响。

### 切片 5 — FilterBar 窄屏 + SearchTable 移动列（#61）

1. `FilterBar`：字段 `w-44`/`min-w-44` → `w-full sm:w-44`（窄屏全宽）；按钮组窄屏 `w-full justify-end`。
2. `SearchTable`：`ColumnDef` 新增 `hideOnMobile?: boolean`（渲染时 `<md` 隐藏该列 + 对应单元格）。additive。
3. 文档：移动列策略（首列 `fixed:"left"` 冻结 + `hideOnMobile` 隐藏次要列 + 卡片列表备选）。

### 切片 6 — 弃用标注 + 选型表 + recipes + 栅格文档（#60/#61）

1. `MasterListTemplate` / `MasterEditTemplate`：JSDoc `@deprecated` → PageChrome list+ListPageShell / form。
2. `docs/component-decision-table.md`：§3.2 补选型表（组织 HR→OrgAdminPage / 分类科目→TreeCrudPage / 左树筛右表→MasterDetailLayout+List / 纯表→ListPageShell）；§2 标 MasterListTemplate 弃用；§6 补 `chromeBorder` 说明。
3. 单页「Admin page recipes」MDX：8 条范式 + 每条 Don't（桌面+移动）。
4. `Row/Col` 文档：写明「span 在 xs 不自动堆叠」，给 `xs={24} md={17}` 示例；`QuickEntryGrid` 极窄容器说明。

## 风险与注意

- `chromeBorder` 默认 `true` 非破坏，但消费方仍需主动传 `false` 才能去双线——在 recipes/选型表强提示。
- `SearchTable.hideOnMobile` 需同时隐藏 header 与 body cell；注意 `fixed` 冻结列与隐藏列共存时的列顺序。
- OrgAdmin 移动 `mobileView` state 需与受控 `selectedId` 协同：选中→detail，返回→清空选中（若受控，调 `onSelect(undefined)`）。
- 不改 `MasterDetailTabs` 公共 API，仅补高度链。
- 1.13.0 minor：行为默认更合理 + 文档；无 breaking，无需 major。
