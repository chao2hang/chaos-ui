# Ecology Wea* 对齐组件波次 (#62–#69) 设计计划

## 概述

消费方 qxy-mop 整仓从泛微 Ecology 迁移，集中提出 8 个对齐 Issue（#62–#69）。本计划一次性补齐「表单级浏览按钮 / 时间增强 / JSON 表单 / 消息中心 / 报表列表 / 左树右栏 / 分页一致 / 反馈 API」八大缺口，目标版本 **1.14.0**。所有新件优先复用现有 ui + business 组合，遵守 Component Reuse Protocol，不重复造轮子。

## 背景与动机

- Ecology 体系是「配置/元数据驱动」：后端吐 JSON → 前端按 schema 渲染表单/列表/浏览。chaos-ui 目前只有碎片能力（BrowseDialog 弹层非字段、FormDesignerRuntime schema 薄、NotificationCenter 无 push、ProTable 无 summary）。
- 消费方迁移 ~50 页，若库不补齐，消费方会各自魔改 → 库失去标准。预期：本波次后消费方有「浏览字段 / JSON 表单 / 报表列表 / 消息推送 / 反馈」唯一答案。
- 同时顺手修同族隐患（FilterBar 高级展开、分页两套、message 与 MessageCenter 职责不清文档缺失），减少二次提 Issue。

## 现有组件检索（强制，防重复造轮子）

对照 `docs/component-decision-table.md` 与 `AGENTS.md` → Component Reuse Protocol。

- [x] 已查选型表：picker / form / table / layout / feedback 各行
- [x] 已检索近名组件：`BrowseDialog` / `EmployeePicker` / `FormDesignerRuntime` / `DynamicFormBuilder` / `NotificationCenter` / `message` / `ProTable` / `SearchTable` / `Pagination`(ui) / `DatePicker` / `MasterDetailLayout` / `TreeView`
- [x] 结论：
  - #62 BrowserField → **新建**公共组件（新 API：字段壳 + id/label 契约），**复用 BrowseDialog** 作为弹层，不双实现
  - #63 DateTimePicker/PeriodPicker → **新建** ui 件，复用 DatePicker 的 format/parse + Calendar
  - #64 SchemaForm → **新建**公共组件（新 schema 协议），内部复用 Input/Select/DatePicker/BrowserField；FormDesignerRuntime 保留，文档引导迁移
  - #65 MessageCenter → **新建**公共组件，**复用 NotificationCenter** 铃铛+面板，加 tabs/loadMore + `useMessageCenter` push store；不复制 NotificationCenter UI
  - #66 ReportTable → **新建**公共组件，**复用 ProTable**，加 report 列元数据协议 + summary 行 + 服务端 loadData
  - #67 左树右栏 → **扩展现有** MasterDetailLayout（resizable），不新建布局件
  - #68 Pagination → **新建** business `PaginationBar`（button 原子，onClick），文档对齐 ui `Pagination`；FilterBar 加 collapsible
  - #69 反馈 API → **文档为主**（message 已有 loading/promise/destroy/key），加 `notify` 协调助手
- [x] **不要做**：不新建 `TreeDetailLayout`；不移植 Ecology browser.xml/SQL 引擎；不接 WebSocket 传输层；不改 NotificationCenter 既有 props 语义（仅 additive）

## 关键决策

| 决策点                              | 选择                                               | 理由                                                     |
| ----------------------------------- | -------------------------------------------------- | -------------------------------------------------------- |
| BrowserField 值契约                 | `value: string\|string[]` + `labels` 回显          | 对齐 WeaBrowser id+replaceDatas；id 为字符串便于多选分隔 |
| BrowserField 弹层                   | 复用 BrowseDialog，内部管 open                     | 不双实现；type 注册表可选，库不绑后端                    |
| SchemaForm 字段类型                 | 含 browser/treeSelect/upload/custom                | 对齐 WeaFormDataItem inputType 子集；custom 透传 render  |
| SchemaForm 联动                     | `dependencies: {name,when,then}`                   | 声明式，对齐 selectLinkageDatas                          |
| MessageCenter vs NotificationCenter | MessageCenter 复用 NotificationCenter + push store | 同职责不分裂；NotificationCenter props additive          |
| ReportTable                         | 包装 ProTable + summary                            | 不另起表格引擎；列元数据是 ProColumn 超集                |
| resizable 实现                      | pointer 事件 + CSS width                           | 无新依赖；min/max 约束                                   |
| PaginationBar                       | button onClick 原子                                | 表格非路由场景；ui Pagination 留给路由                   |
| notify.inbox                        | 调 useMessageCenter.push                           | toast 与 inbox 职责分离                                  |
| 版本                                | 1.14.0（minor，新增公共 API）                      | 多个新公共组件，非 patch                                 |

## 切片清单

### S1 — #62 BrowserField（P0）

**文件**：`components/business/browser-field.tsx`（+ test）、barrel `components/business/index.ts`

**API**：

```ts
interface BrowserFieldProps {
  type?: "employee" | "department" | "company" | "product" | "custom";
  value?: string | string[];
  labels?: { id: string; label: string }[];
  onChange?: (value: string | string[], items: BrowseItem[]) => void;
  multiple?: boolean;
  disabled?: boolean;
  clearable?: boolean;
  placeholder?: string;
  size?: "sm" | "default";
  // 弹层透传
  loadData?: BrowseDialogProps["loadData"];
  columns?: BrowseColumn[];
  tree?: BrowseDialogProps["tree"];
  items?: BrowseDialogProps["items"]; // 本地静态
  rowKey?: string;
  pageSize?: number;
  title?: string;
  // 联想
  complete?: (keyword: string) => Promise<BrowseItem[]>;
  // 新增
  onAdd?: () => void | Promise<BrowseItem | void>;
  // 展示
  linkHref?: (item: BrowseItem) => string;
  className?: string;
}
```

- 触发器：outline button 形如 Input，左侧标签 chips（多选）/ 文案（单选），右侧放大镜图标 + 可选「+」新增 + 清空 ×
- 点击触发 → 内部 `open` → 渲染 `<BrowseDialog>`（selectionMode 由 multiple 决定）
- confirm → 提取 `{id, label}` → `onChange(value, items)`
- `registerBrowserType(type, {loadData, columns, ...})` 可选注册表（库不绑后端）
- 尺寸 sm 与 FilterBar/Input sm 同高（h-7）

**验收**：单选员工 / 多选部门树 / 本地 items / 新增按钮回填；FormGrid 内高度对齐 Input sm

### S2 — #63 DateTimePicker / PeriodPicker / datePresets（P1）

**文件**：`components/ui/date-time-picker.tsx`、`components/ui/period-picker.tsx`、`components/ui/date-presets.ts`（+ tests）、ui barrel

### S3 — #64 SchemaForm（P0）

**文件**：`components/business/schema-form.tsx`（+ test）、barrel

**协议**：

```ts
type FieldType =
  | "text"
  | "textarea"
  | "number"
  | "select"
  | "multiSelect"
  | "checkbox"
  | "switch"
  | "radio"
  | "date"
  | "dateTime"
  | "dateRange"
  | "browser"
  | "treeSelect"
  | "upload"
  | "custom";
interface FormFieldSchema {
  name;
  label;
  type;
  required?;
  hidden?;
  disabled?;
  placeholder?;
  defaultValue?;
  options?;
  browser?;
  rules?;
  colSpan?;
  dependencies?;
  props?;
  render?;
}
interface FormSchema {
  groups?: { key; title?; collapsed?; fields }[];
  fields?: FormFieldSchema[];
}
```

- `SchemaForm({ schema, value, onChange, onSubmit, layout? })`
- 分组 → Card section（title + 可折叠）；无分组 → 单列
- 按 type 渲染：text/textarea/number→Input/Textarea；select/multiSelect→Select/MultiSelect；date→DatePicker；dateTime→DateTimePicker；browser→BrowserField；custom→`field.render`
- `dependencies`：监听依赖字段值，`when` 匹配则合并 `then`（隐藏/清空/改选项）
- 校验：`rules`（required/pattern/min/max/custom）→ 错误文案红字
- 复用 `FormGrid`/`LabeledField` 布局；colSpan 控制跨列

**验收**：JSON 渲染 文本+下拉+日期+browser；联动选公司→清空部门；迁移说明文档

## 风险 / 开放问题

- BrowserField 与既有 `*Picker`（EmployeePicker 等）职责重叠 → 文档明确：Picker=combobox 轻量单选；BrowserField=弹层+多选+tabs+树。选型表区分。
- SchemaForm 与 FormDesignerRuntime 双轨 → 文档迁移说明，不立即废弃 Runtime（标 `@deprecated` 指向 SchemaForm）。
- ReportTable 与 ProTable 重叠 → ReportTable 是 ProTable 的「服务端报表协议」封装，选型表说明何时用哪个。
- resizable 拖拽与 Tailwind v4 响应式变量交互 → 仅桌面 md+ 启用，移动禁用。
