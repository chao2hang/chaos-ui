# 业务工具栏 size 对齐 (#46 FilterBar + #47 扩展面)

## 概述

统一列表/对话框工具栏里 **输入控件与按钮的高度档**，消除 default `h-8` 与 `Button size="sm"`（`h-7`）同排「一高一矮」。

- **#46**：`FilterBar` 首例（P2，全站列表可见）
- **#47**：同模式扫描清单（CrudToolbar / FilterBuilder / AdvancedDataTable / DictManageDialog / InvoiceTitlePicker / AuditTrailDiff）

**推荐默认统一为 sm（h-7）**，贴合现有工具栏按钮密度；可选 bar 级 `size` prop。

建议 **patch 或 minor**（若公开 `FilterBar.size` 为新 API → minor 更稳）。

## 背景与动机

- 原语 size 波（#28–#32）已让 Input / SelectTrigger / NativeSelect / Button 支持 `sm | default`。
- 业务组合仍混用：字段 default、操作钮 sm。
- 消费方 FE-11：不本地 CSS 掩盖。

## 高度阶梯（库内约定）

| 档      | Button  | Input | SelectTrigger             | NativeSelect |
| ------- | ------- | ----- | ------------------------- | ------------ |
| default | default | `h-8` | `data-[size=default]:h-8` | `h-8`        |
| sm      | `h-7`   | `h-7` | `h-7`                     | `h-7`        |

### 明确不改（issue 边界）

- 远程选择器列表内 ghost sm 选项行

## 设计方案

### 策略 A（推荐，本计划默认）

工具栏密度 **统一 sm**：

- 所有工具栏行内 Input / SelectTrigger / NativeSelect 使用 `size="sm"`
- 去掉硬编码 `className="h-8 …"`，改为 `size="sm"` + 布局类（`pl-8`、`w-44`）
- 按钮保持 `size="sm"`

### 策略 B（备选）

统一 default：按钮改 `size="default"`。**不推荐**——列表密度会变「胖」，与大量已 sm 的工具栏冲突。

### 可选 API（FilterBar 优先）

```ts
// FilterBarProps
size?: "sm" | "default"; // default: "sm"
```

- 下发给 Input / SelectTrigger / NativeSelect / action Buttons
- #47 组件可后续复用同一默认，或本波次直接写死 sm（更少 API 面）

**本计划建议：**

1. FilterBar：实现 `size` prop，默认 `"sm"`（新默认相对 1.6.x 字段高度会 **变矮 4px** — 见风险）
2. #47 各组件：本波次直接对齐 sm，不强制公开 toolbar size（除非已有 size API）

### 视觉变化说明（诚实）

当前 FilterBar 字段是 h-8、按钮 h-7。统一 sm 后：

- 字段：32px → 28px
- 按钮：不变

这是 **有意视觉调整**（非 pure bugfix）。若必须零视觉变化，只能走策略 B（按钮变高），产品上更差。建议 CHANGELOG 写在 **Changed**，patch/minor 由维护者选（倾向 **patch** 若视为 density 修正，或 **minor** 若视为 toolbar 契约）。

## 风险与注意

- **默认变矮**：FilterBar 字段从 h-8→h-7 影响全站列表；需 CHANGELOG 明示；消费方勿用 CSS 拉回。
- 硬编码 `h-8` 与 `size` 并存时 tailwind 可能冲突 — **删除 h-8**，只留 size。
- `custom` field `render` 无法强制 size — 文档注明自定义字段自行对齐。
- 与 #41 Select portal 无冲突；只改 size。
