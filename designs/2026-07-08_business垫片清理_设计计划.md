# Business 垫片清理 设计计划

## 概述

删除 `components/business/` 下 9 个向后兼容 shim 文件，将所有引用统一指向 `components/ui/`。

## 背景与动机

- 9 个本属于 UI 类的组件（combobox, empty-state, fab, rating, segmented-control, time-picker, tour, watermark, transfer）的实现在 `components/ui/`，`components/business/` 中仅保留一行 re-export shim
- 这些 shim 造成维护负担（双份路径、混淆归属），且无实际功能价值
- 清理后路径清晰：所有 UI 组件统一从 `@/components/ui/` 导入

## 设计方案

### 技术方案

纯机械路径替换，不涉及任何运行时行为变更：

- `@/components/business/{name}` → `@/components/ui/{name}`
- `./{name}` → `@/components/ui/{name}` (测试文件中)
- 从 barrel 导出中移除 shim 行
- 删除 9 个 shim 源文件

### 影响文件统计（77 个）

| 类别 | 文件数 | 操作 |
|------|--------|------|
| Shim 源文件 | 9 | 删除 |
| Barrel 导出 | 2 | 移除 9 行 |
| 测试文件 | 7 | 改 import |
| 业务组件 | 1 | 改 import |
| Package 内部 | 4 | 改 import |
| Story 文件 | 12 | 改 import |
| MDX 文档 | 36 | 改 import + 代码示例 |
| Docs 组件 | 4 | 改 import |
| 元数据 | 2 | 改路径 |
