# Story 文件类型错误修复 — 第一阶段设计

## 概述

修复 `apps/docs/src/` 下 552 个 `.stories.tsx` 文件中由 `satisfies Meta<typeof C>` 模式导致的 ~360 个 TS2322 类型错误（`args: {}` 和 `render` 模式类型不匹配），以及约 38 个其他类型错误（TS2353、TS2741、TS2352、TS2305 等）。

## 背景与动机

- 当前 `tsc --noEmit` 在 `apps/docs` 中报 1278 个错误
- 约 62% 来自 Story 文件（300 个 unique 文件）
- 核心根因：`const meta = { ... } satisfies Meta<typeof C>` + `StoryObj<typeof meta>` 产生 `Partial<Props> & Props` 的 args 类型，导致所有属性变为必填
- 替换为 `const meta: Meta<typeof C> = { ... }` 后 `StoryObj` 能正确推导 `Partial<Props>`

## 方案

### 批量替换：`satisfies Meta<` → `: Meta<`

将 552 个文件的 meta 声明从：

```typescript
const meta = {
  title: "...",
  component: SomeComponent,
  tags: ["autodocs"],
} satisfies Meta<typeof SomeComponent>
```

改为：

```typescript
const meta: Meta<typeof SomeComponent> = {
  title: "...",
  component: SomeComponent,
  tags: ["autodocs"],
}
```

### 实现方式

- 正则：`const meta = {\n([^}]*\n)*\} satisfies Meta<typeof (\w+)>`
  用 `const meta: Meta<typeof $1> = {$2}`
- 用 `sed` 或单次写入脚本执行

### 验证方式

1. 运行 `npx tsc --noEmit` 检查 story 错误数量变化
2. 运行 `npm run build-storybook` 确保 Storybook 可构建
3. 运行 `npm run dev` 确认 docs 网站可启动

### 手工修复项（~38 errors）

替换后剩余 story 专属错误：

| 错误类型 | 数量 | 修复方式 |
|---------|------|---------|
| TS2353 多余属性（如 `activities`→`items`） | ~19 | 重命名 args 属性 |
| TS2741 缺少属性 | ~8 | 补全缺失参数 |
| TS2352 类型转换（as 转换） | ~3 | 改为双层 `as unknown as` |
| TS2305/TS2724 导出名不匹配 | ~5 | 修正导入名（如 BOMTreeEditor）|
| TS18046 `unknown` 不可赋值 | ~6 | 加类型断言或补充类型 |

### 其他阶段

Phase 1 仅修复 Story 文件（当前阶段）。
Phase 2 处理 Business Shim 文件（~94 errors，TS2484/TS2430/TS2303 等）。
Phase 3 处理模块缺失和其他错误（~86 errors，`component-story-previews.tsx`）。

## 预期结果

- 消除 ~360 个 TS2322，story 错误从 ~404 降至 ~38
- 所有 247 个 `args: {}` 故事和 420 个含 `render` 的故事通过类型检查
- Storybook 构建保持正常

## 不做的范围

- 不改动 `component-story-previews.tsx`

- 不改动业务组件源码
