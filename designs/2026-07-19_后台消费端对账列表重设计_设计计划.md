# 后台消费端对账列表重设计

## 概述

重做文档站后台模板中的消费端列表场景，使其体现成熟 ERP 的信息层级、筛选效率和响应式行为，并与 Chaos UI 现有主题 token、AdminShell 和列表组件保持一致。设计基准为用户提供的“费用管理 / 对账签认”截图，落地入口为 `/examples/admin` 的后台预览。

## 背景与动机

- 当前后台模板仍是通用的“Chaos Admin + 卡片占位 + 原生表格”演示，无法展示消费方实际的密集列表工作流。
- 侧栏层级、顶栏导航、页级筛选和表格动作缺少统一的信息密度，桌面空间利用率低，移动端也没有清晰的退化路径。
- 需要一个可交互、可复用现有组件的标杆场景，帮助消费方直接参考成熟的后台页面组合方式。

## 设计方案

### 视觉与交互方向

- 采用 Chaos UI 的中性白底、细边框、品牌蓝和黑色主操作色；不引入独立主题色或渐变。
- 桌面采用固定 240px 侧栏、64px 顶栏、紧凑多级导航和内容区 24px 横向节奏；内容采用单一主工作区，不套多层卡片。
- 顶栏包含品牌标识、面包屑、通知和用户菜单；内容首屏以标题/状态摘要、筛选工具栏和可横向滚动的表格为主。
- 表格金额右对齐、关键应付净额加粗；状态用语义 badge；待签认行提供明确主动作，已关闭/已处理行只显示次要明细动作。
- 筛选支持账期输入、状态选择、查询、重置和刷新；新增、签认和详情动作更新本地演示数据并给出可见状态。
- 小屏保留顶栏和主列表，侧栏由 `AdminShell` 的移动抽屉承载；筛选控件自动堆叠，表格保持最小可读宽度并允许横向滚动。

### 设计 token（复用全局）

| 类别     | 取值                                        |
| -------- | ------------------------------------------- |
| 背景     | `bg-background` / `bg-muted/30`             |
| 表面     | `bg-card`                                   |
| 文本     | `text-foreground` / `text-muted-foreground` |
| 品牌     | `text-brand-600`、`bg-brand-50`             |
| 边框     | `border-border`                             |
| 圆角     | `rounded-md`（不超过库默认 10px）           |
| 阴影     | `shadow-xs`，仅用于浮层/活动状态            |
| 间距     | 4/6/8/12/16/24px，使用 Tailwind token       |
| 控件密度 | 筛选和表格动作优先 `size="sm"`              |
