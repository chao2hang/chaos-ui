# 开放 Issue 波次 #70/#71/#72 设计计划

## 概述

本波次处理当前仓库的三个开放 issue，并作为一个非破坏性 minor release 交付：

- #72 BrowserField 预查联想与 WeaBrowser completeUrl 体验对齐；
- #71 ListPageShell 工具条与 SearchTable 表格边框水平 inset 对齐；
- #70 Row/Col 响应式 Tailwind 栅格规则可用性修复。

## 背景与动机

- #72 阻塞消费方通过 registry 统一接入联想、已选再输、键盘选择和高级浏览双通道。
- #71 同一列表页的工具条和表格使用不同水平坐标系，桌面端产生固定 1rem 视觉错位。
- #70 运行时拼接的响应式 class 无法被 Tailwind 静态扫描，导致 `md` 等断点布局失效。

## 现有组件检索（强制，防重复造轮子）

对照 [`docs/component-decision-table.md`](../docs/component-decision-table.md) 与 `AGENTS.md` → Component Reuse Protocol。

- [x] 已查选型表：BrowserField/BrowseDialog、ListPageShell/SearchTable、Row/Col 均已有官方入口。
- [x] 已用 `rg` 检索实现、测试、Story、CSS 入口及 `--card-spacing`/栅格 class。
- [x] 已打开相关 Story：`src/stories/business/BrowserField.stories.tsx` 及布局/列表相关测试。
- [x] 结论：复用现有组件，仅扩展 BrowserField 非破坏性 props/registry 配置并修复现有布局/CSS。
- [x] 不新建近名 picker、table shell 或 grid 组件；不使用 deprecated 公共件。

## 设计方案

### 技术方案

1. BrowserField：registry 支持 `complete`、`completeMinChars`、`completeDebounceMs`、`completeAdvancedSearch`；props 优先于 registry。单选有值时保留可聚焦输入态，增加 listbox/option 键盘高亮、Enter 选中、Escape/Tab 收口和高级搜索页脚；BrowseDialog 受控传入当前选择；用序号丢弃过期请求。
2. ListPageShell：工具条 actions 在两种 placement 下都补 `pr-[var(--card-spacing,1rem)]`，窄屏保持现有换行布局。
3. Row/Col：保留稳定的公开 API，改由 CSS custom properties 输出 span、offset、order，再由包内 `.chaos-col` 与 `sm` 至 `2xl` 媒体规则解释；不再依赖 Tailwind 扫描运行时 class，补组件与视觉回归测试。

### 接口设计

```typescript
interface BrowserTypeConfig<T extends BrowseItem = BrowseItem> {
  complete?: (keyword: string) => Promise<T[]>;
  completeMinChars?: number;
  completeDebounceMs?: number;
  completeAdvancedSearch?: boolean;
  // existing browse configuration remains unchanged
}

interface BrowserFieldProps<T extends BrowseItem = BrowseItem> {
  completeMinChars?: number;
  completeDebounceMs?: number;
  completeAdvancedSearch?: boolean;
}
```

All new props are optional and preserve the no-`complete` behavior from 1.15.x.
