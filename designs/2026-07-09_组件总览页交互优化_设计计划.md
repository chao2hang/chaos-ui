# 组件总览页交互优化

## 问题诊断

当前 `/components` 页面（`apps/docs/app/components/page.tsx` + `component-search.tsx`）存在三个核心痛点：

| #   | 问题                                                               | 严重度 |
| --- | ------------------------------------------------------------------ | ------ |
| 1   | Business 分类有 **339 个组件**，单分类滚到底需要滑动 20+ 屏        | 🔴 高  |
| 2   | 没有左侧导航，在 8 个分类之间跳转需要依赖 sticky tab 栏 + 手动滚动 | 🟡 中  |
| 3   | 搜索没有 `Cmd+K` 快捷键，无法像 Linear/Vercel 那样"瞬搜瞬跳"       | 🟡 中  |

### 当前架构速览

```
┌─────────────────────────────────────────────┐
│  Hero (section) — 标题 + 8 个分类统计卡片    │
├─────────────────────────────────────────────┤
│  Search (input) — 搜索框 + 命中计数          │
│  Category Tabs (sticky) — General / Layout..│
│                                             │
│  ▼ General    (25 cards)                    │
│  ▼ Layout     (11 cards)                    │
│  ▼ Navigation (15 cards)                    │
│  ▼ Form       (35 cards)                    │
│  ▼ DataDisplay(28 cards)                    │
│  ▼ Feedback   (18 cards)                    │
│  ▼ Business   (339 cards) ← 核心问题        │
│  ▼ SystemLayout(25 cards)                   │
└─────────────────────────────────────────────┘
```

---

## 优化方案

### 方案对比

| 方案                                        | 改动量 | 效果          | 风险 |
| ------------------------------------------- | ------ | ------------- | ---- |
| A: 轻量增强（Cmd+K + 折叠 + 子分类）        | ~2h    | 解决 80% 痛点 | 低   |
| B: 右侧边栏 + 树形导航                      | ~4h    | 解决 95% 痛点 | 中   |
| C: 全量重构（左侧栏 + 命令面板 + 虚拟滚动） | ~1d    | 终极方案      | 高   |

**推荐：方案 A → B 渐进实施。** 先用 A 快速止血，再视情况升级到 B。

---

## Phase 1: 轻量增强（推荐优先实施）

### 1.1 — 添加 Cmd+K 命令面板

Linear / Vercel 的核心交互模式。在任何位置按 `Cmd+K`（Windows: `Ctrl+K`）弹出搜索浮层。

**实现要点：**

- [ ] 新建 `@/components/command-palette.tsx`（"use client"）
  - 监听 `keydown`：`Meta+k` / `Ctrl+k` 打开
  - 半透明遮罩 + 居中输入框（Linear 风格）
  - 实时过滤，结果列表支持键盘 ↑↓ 导航 + Enter 跳转
  - 按 Escape 关闭
- [ ] 将搜索逻辑从 `component-search.tsx` 提取为公共 hook `useComponentSearch`
- [ ] 在 `app/layout.tsx` 中挂载 `<CommandPalette />`

**效果：**

- 按 Cmd+K → 输入 "direction" → Enter → 直接跳到 Direction 详情页
- 无需滚动、无需鼠标

---

### 1.2 — Business 分类添加子分组

339 个组件的根本解决方案：按业务域拆分子分组。

**子分组规则（基于现有组件名前缀分析）：**

| 子分组    | 匹配前缀                             | 预估数量 |
| --------- | ------------------------------------ | -------- |
| 仪表盘    | Dashboard*, Metrics*                 | ~30      |
| 报表      | Report*, Chart*, Stat*               | ~40      |
| 审批      | Approval*, Workflow*, Audit*         | ~35      |
| 列表页    | List*, Table*, Grid*, CRUD*          | ~50      |
| 详情页    | Detail*, Form*, Profile*, Card*      | ~45      |
| 弹窗/抽屉 | Modal*, Drawer*, Dialog*, Popup*     | ~35      |
| 工具栏    | Toolbar*, ActionBar*, Header*        | ~25      |
| 输入/选择 | Picker*, Selector*, Input*, Chooser* | ~30      |
| 其他      | 不匹配以上                           | ~50      |

**实现要点：**

- [ ] 在 `components.meta.ts` 中为 Business 分类组件增加 `subCategory?: string` 字段
- [ ] 在 `component-search.tsx` 中 Business section 内按子分组渲染子标题
- [ ] 子分组标题用 `<h3>`，视觉层级低于主分类 `<h2>`
- [ ] 只显示有内容的子分组

**效果：**

- 339 → 分成 9 个子区块，每区块约 30-50 个组件
- 视觉上仍然是"一页"，但逻辑上可定位了

---

### 1.3 — 分类区块可折叠

每个分类 section 添加折叠/展开按钮。默认全部展开。

**实现要点：**

- [ ] 在 `component-search.tsx` 中给每个 `<section>` 增加折叠状态
- [ ] 用 `useState<Set<Category>>` 管理展开/折叠状态
- [ ] 折叠时保留分类标题 + 数量，隐藏卡片网格
- [ ] 添加"全部展开 / 全部折叠"按钮在 category tabs 右侧

**效果：**

- 用户可以把不需要的分类收起来，只浏览目标分类
- 大幅减少页面高度
