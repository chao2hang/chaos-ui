# 全项目多角色审计整改 设计计划

## 概述

对 Chaos UI 进行多角色只读审计（架构/API、组件复用与组合、样式与 a11y、公共方法与 hooks、测试/Story/文档、工程治理与发布、运行时缺陷），按推荐范围实施 **P0/P1 + 证据充分的高价值 P2**，验证后提交、推送并发布 minor 版本。

## 设计方案

### 原则

1. **契约优先**：类型/Story 承诺的行为必须实现或从公开 API 删除。
2. **最小安全修复**：不改无关 API 形状；必要时 additive。
3. **验证窄测 → 相关 suite → typecheck/lint → release:check**。
4. **发布**：minor（审计修复波次 + 行为修正），CHANGELOG 列审计项与破坏性说明（若有 duration 行为修正）。

### 技术方案（关键契约）

- **message.duration**：保持「秒」公共文档，内部 `* 1000` 传 sonner；`duration: 0` 仍表示不自动关闭。
- **ApiClient 401**：`config._retry` 只刷新一次；并发 401 尽量 single-flight（最小实现：retry 标记即可）。
- **FilterBar**：i18n via `useSafeTranslation`；实现 date-picker / date-range-picker 或组合现有 DatePicker/DateRangePicker；reset 回到 defaultValue 初始表；保留 Select `null` 修复。
- **BrowserField**：外壳改为非 nested button 结构。
- **SchemaForm**：初始化合并 field.defaultValue；browser 字段并行维护 labels。
- **PeriodPicker**：内部 committed state。
- **DateTimePicker**：展示/提交时 minute 对齐 step。
- **Pagination**：`pageSize = Math.max(1, pageSize || 1)`。
- **a11y modals**：ImageViewer / MobileShareSheet 补 dialog 语义 + Escape + 基础 focus。
- **AdvancedDataTable**：去掉 `export *`，仅保留 deprecated named export。
