# 树与主从布局 #51 / #53 / #54 设计计划

## 概述

修复 OrgTree 同级缩进（#51）、MasterDetail 桌面固定栏宽（#53）、TreeView 选择模式（#54）。

## 现有组件检索

- [x] OrgTree 独有叶子 +16 缩进；TreeView/TreeSelect/FileManager 无此问题
- [x] MasterDetail 仅 maxWidth+md:w-auto；FileManager/AppShell 已固定 width
- [x] TreeView 无 selectionMode；OrgTree/TreeSelect/TreeTable 已有模式
- [x] 结论：修既有组件 API/样式，不新建树/布局组件

## 技术方案

1. **#51** 缩进槽 `depth * 20`，删除叶子额外 16；叶子保留 `w-5` 占位
2. **#53** 桌面 open：`width`+`maxWidth`=sidebarWidth；去掉内容驱动 `md:w-auto`；内层 `min-w-0`
3. **#54** `selectionMode?: "single" | "multiple"` 默认 `"multiple"`（兼容）；single 再点清空；OrgTree single 同步再点清空
