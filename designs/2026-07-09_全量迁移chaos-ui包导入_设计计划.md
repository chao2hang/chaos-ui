# 全量迁移至 @chaos_team/chaos-ui 包导入 设计计划

## 概述

将 `apps/docs/` 文档站中所有从本地 `@/components/*`、`@/hooks/*`、`@/lib/*` 路径导入的组件、hooks 和工具函数，全量替换为从 `@chaos_team/chaos-ui` 包（v1.1.0）导入。同时清理本地重复的组件副本，引入包级 CSS，统一页面结构与布局。

## 背景与动机

- **版本升级**：`@chaos_team/chaos-ui` 已从 1.0.5 → 1.1.0
- **当前问题**：文档站（`apps/docs/`）维护了一份与包源码完全重复的组件副本（~500+ 文件在 `apps/docs/@/components/`），导致：
  - 组件更新需双向同步，极易遗漏
  - 文档站可能展示过期版本的组件行为
  - 维护成本高、构建体积膨胀
- **预期效果**：文档站直接消费 `@chaos_team/chaos-ui` 包，消除重复代码，确保展示的始终是最新发布版本

## 设计方案

### 技术方案

1. **构建包**：执行 `pnpm run build:pkg` 生成 `dist/` 目录
2. **添加依赖**：在 `apps/docs/package.json` 中添加 `"@chaos_team/chaos-ui": "file:../.."`
3. **Next.js 配置**：在 `apps/docs/next.config.ts` 中添加 `transpilePackages: ["@chaos_team/chaos-ui"]`
4. **导入路径映射规则**：

| 原始导入路径                  | 替换为                          | 说明                   |
| ----------------------------- | ------------------------------- | ---------------------- |
| `@/components/ui/xxx`         | `@chaos_team/chaos-ui/ui`       | UI 组件 barrel 导入    |
| `@/components/business/xxx`   | `@chaos_team/chaos-ui/business` | 业务组件 barrel 导入   |
| `@/components/layout/xxx`     | `@chaos_team/chaos-ui/layout`   | 布局组件 barrel 导入   |
| `@/components/mobile/xxx`     | `@chaos_team/chaos-ui/mobile`   | 移动端组件 barrel 导入 |
| `@/hooks/xxx`                 | `@chaos_team/chaos-ui/hooks`    | Hooks barrel 导入      |
| `@/lib/xxx`                   | `@chaos_team/chaos-ui/lib`      | 工具库 barrel 导入     |
| `@/components/site-header` 等 | 保持 `@/components/xxx`         | 文档站专用组件不动     |

5. **CSS 引入**：在 `apps/docs/app/layout.tsx` 中添加 `import "@chaos_team/chaos-ui/styles.css"`
6. **清理重复文件**：迁移完成后删除 `apps/docs/@/components/ui/`、`apps/docs/@/components/business/`、`apps/docs/@/components/layout/`、`apps/docs/@/components/mobile/`、`apps/docs/@/hooks/`（保留文档站专用组件和 docs-app-specific business 组件）

## 风险与注意

1. **Barrel 导入命名冲突**：从 barrel 导入时可能出现同名导出冲突，需逐个排查
2. **docs-app-specific 组件**：部分组件只存在于 docs app 中（如 `chart-empty.tsx`, `coach-mark.tsx` 等），不能迁移到包导入
3. **CSS 优先级**：包 CSS 与文档站 CSS 可能存在优先级冲突
4. **类型导出**：`import type` 语句也需要正确迁移
5. **动态导入**：`dynamic(() => import(...))` 语句需特殊处理
6. **MDX 文件**：MDX 中的导入语句也需要迁移
