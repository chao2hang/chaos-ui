# Form 控件 size 波次（#29–#32）设计计划

## 概述

在 TreeSelect `size`（#28）落地后，按同一契约为其余表单触发器补 `size?: "sm" | "default"`，使 PageHeader / 工具栏场景可与 `Button size="sm"`、`SelectTrigger size="sm"`（`h-7`）并排对齐。

本计划为 **波次总览 + 分 issue 实施清单**，每个 issue 独立 PR，不捆绑。

## 背景与动机

- 工具栏/筛选条中多控件与 sm 按钮并排时高度不一致（`min-h-8` / `h-8` / `min-h-9` 混用）。
- #28 已锁定契约：`size` + 根/触发器 `data-size`；sm 单行 `h-7 min-h-7`；多值仅 `min-h-7`、不硬裁切；sm 时 compact Badge（可选）。
- 相关 issue：#29 UserBrowse / DepartmentBrowse；#30 Cascader / OrgTreeSelect / AddressPicker；#31 Combobox / MultiSelect / TagsInput；#32 Input（P3）。

## 设计方案

### 共享契约（各 issue 必须遵守）

| 项        | 约定                                                                 |
| --------- | -------------------------------------------------------------------- |
| API       | `size?: "sm" \| "default"`，默认 `"default"`，additive、非破坏       |
| 默认高度  | 保持各组件现有 default 高度（不借机改默认）                          |
| sm 单行   | 对齐 `h-7` / `min-h-7`，圆角与 SelectTrigger sm 一致倾向             |
| sm 多值   | 仅 `min-h-7`（允许 Badge 换行增高），禁止固定 `h-7` 裁切             |
| data-size | 根或主触发器打 `data-size={size}`                                    |
| Badge     | sm 可用紧凑 class；不改全局 Badge API                                |
| 测试      | 至少：默认 `data-size`、`size="sm"`、`h-7`/`min-h-7` class（适用时） |
| Story     | 与 Button sm 并排或 PageHeader actions 场景                          |

可选后续（非本波次门禁）：抽取 `formControlSizeClasses(size)` helper，减少重复 class 字符串——**先复制 #28 模式，helper 仅在第二/三波确认收益后引入**。

### 分 issue 范围

#### #29 UserBrowse / DepartmentBrowse（建议优先）

- 现状：触发器 `min-h-8` 类硬编码。
- 交付：两组件 `size` prop + sm 高度 + stories/tests。
- 不改远程检索契约（见独立「浏览 / 选人远程检索接口」计划）。

#### #30 Cascader / OrgTreeSelect / AddressPicker

- 现状：触发器多为 `h-8` / 固定高度，无 size。
- 注意：`OrgTreeSelect` 为独立 Popover 实现（**不是** TreeSelect 包装），需单独改。
- AddressPicker 若为复合输入，sm 对齐主触发区即可。

#### #31 Combobox / MultiSelect / TagsInput

- 现状：MultiSelect 默认 `min-h-9` 偏高；Combobox / TagsInput 缺 size。
- 风险：MultiSelect default 保持 `min-h-9`（不借机压到 8），仅 sm 压到 `min-h-7`。
- TagsInput 与多 Badge 交互同 TreeSelect multi 规则。

#### #32 Input size（P3）

- 规范级：与既有 Input 文档 / R2 对齐；可与 shadcn `h-8`/`h-9`/`h-7` 映射确认后再动。
- 优先级最低；可在 #29–#31 合并后单独开。

## 风险与注意

- 不要在本波次改 TreeTable flush（#27）或 Tabs（#26 已 release）。
- OrgTreeSelect ≠ TreeSelect：禁止假设 prop 透传即可。
- MultiSelect default `min-h-9` 是既有产品选择，sm 才对齐 h-7。
- 消费方（mop 等）升级后需显式传 `size="sm"` 才会变矮。
