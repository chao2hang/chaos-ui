# chaos-ui 消费方手写模式收编计划

## 背景

审计里 6 类「手写」多数**不是缺组件**，而是消费方（qxy-mop/frontend）没把已有 chaos-ui 能力用满，或库缺一层薄胶水。本轮**只在 chaos-ui 补库能力**，不改消费方页面。

| 审计项                            | 库现状                                                      | 本轮动作                                                                                |
| --------------------------------- | ----------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| 表单校验 RHF+zod                  | Form 原语已有；`FormDialog` 藏在 CrudPage 内                | **导出 FormDialog**                                                                     |
| 详情行 Descriptions               | 已有且可用                                                  | 不改（文档/发现性即可）                                                                 |
| 登录页 Spin / 错误卡 / BrandBadge | LoadingPage/FullPageLoader/ErrorPage/BrandBadge 已有        | **PageLoading 别名**（h-dvh 语义）                                                      |
| not-found / error                 | `NotFound`/`InternalError` 已从 `./next` 导出               | 不改（消费方接线）                                                                      |
| 列表脚手架重复                    | FilterBar/SearchTable/CrudPage 有；缺 RQ 列表 hook 与微组件 | **usePageList + RefreshButton + RecordCount + ListPageShell + toSearchTablePagination** |
| 日期 Input                        | DatePicker 已有，仅 `Date`                                  | **valueAsString 模式**                                                                  |

## 设计原则

1. **小而可组合**：优先独立 hook/按钮/工具函数；`ListPageShell` 只做轻量拼装，不替代 `CrudPage`。
2. **双数据栈并存**：`useCrud`（命令式）与 `usePageList`（React Query）并列，不硬合并。
3. **API 贴近 mop 实战**：`usePageList` 以 mop 现有实现为蓝本，去掉 app 内部常量依赖。
4. **先计划文档再写代码**（AGENTS.md 设计看板规范）。

### 1. P0 — `usePageList`（hooks）

**文件**

- 新增 `hooks/use-page-list.ts`
- 新增 `hooks/use-page-list.test.ts`
- 导出：`hooks/index.ts`

**API（从 mop 收编，去 app 耦合）**

```ts
export interface PageListParams {
  page: number
  pageSize: number
  keyword?: string
  sortBy?: string
  sortOrder?: "asc" | "desc"
  [key: string]: unknown
}

export interface PageListResult<T> {
  /** 行数据：兼容 records / list */
  records?: T[]
  list?: T[]
  total: number
}

usePageList<T, P extends PageListParams>({
  queryKey: readonly unknown[]
  fetcher: (params: P) => Promise<PageListResult<T>>
  initialPage?: number        // default 1
  initialPageSize?: number    // default 20
  initialFilters?: Omit<P, "page" | "pageSize">
  staleTime?: number          // default 60_000
  selectRows?: (data) => T[]  // 可选；默认 data.records ?? data.list ?? []
})

// returns
{ data, rows, isLoading, isFetching, page, pageSize, filters,
  setFilters, setPage, setPageSize, refresh, pagination }
// pagination 形状直接可喂 SearchTable
```

**依赖**：peer `@tanstack/react-query`（已有）；内部可用现有 `useQuery` 或直接 `useQuery` from tanstack + `keepPreviousData`。

**测试**：setFilters 重置页码；pageSize 变化重置页 1；pagination.onChange；rows 从 records/list 归一。

valueAsString?: boolean
